Web

Sciences Numériques et Technologie - Seconde

Présentation du cours

Ce cours vous permettra de comprendre les fondements du Web, son histoire, son fonctionnement et les technologies qui le composent. Vous découvrirez comment les pages web sont créées, structurées et comment elles communiquent entre elles.

Point Historique et Définitions

Le WEB a été créé en 1989 au CERN (institut de recherche Suisse) par une équipe chapeautée par Tim Berners-Lee et Robert Cailliau.

Le but du projet initialement était la création d'une application qui permettrait l'échange de données sur Internet.

La technologie du WEB repose sur l'utilisation d'hyperliens. Les hyperliens sont des liens cliquables souvent bleus qui permettent d'accéder à des données qui sont stockées sur des serveurs, comme des pages WEB, des images ou tout type de contenu.

Tim Berners-Lee et Robert Cailliau, créateurs du Web

Pour accéder à toutes ces données, on utilise un navigateur WEB, un logiciel qui permet de traiter des demandes de données (appelées requêtes), les afficher ou réaliser certains traitement sur celles-ci.

Anecdote : La toute première page WEB, celle créée par les chercheurs du CERN, est toujours accessible en ligne.

Modèle Client - Serveur

Dans les échanges sur le Web, deux types de machines communiquent :

Client

Une machine qui souhaite recevoir des informations ou des données. Cette machine correspond à la machine réceptrice.

Serveur

Une machine qui dispose d'informations ou de données et qui a pour rôle de les envoyer. Cette machine correspond à la machine émettrice.

Schéma du modèle client-serveur

Lors d'un échange entre un client et un serveur, ceux-ci émettent des échanges formalisés que l'on appelle requêtes.

Adresse IP et URL

Il peut être fastidieux de retenir toutes les adresses IP de tous les sites que nous connaissons.

On appelle URL (Uniform Ressources Locator) l'adresse d'un site WEB, adresse correspondante à l'adresse IP du serveur où se retrouve la page mais dans des caractères intelligibles par l'être humain.

Structure d'une URL

Une URL est constituée de plusieurs parties séparées par des points:

http(s)://www.google.com
  • http(s):// : protocole utilisé
  • www : sous-domaine
  • google : nom de domaine
  • com : extension du domaine

Le serveur DNS (Domain Name Server) possède une table de correspondance entre l'adresse IP du serveur disposant des informations et d'une adresse dite symbolique que nous pouvons retenir plus facilement.

Fonctionnement du serveur DNS

Le langage HTML

Le HTML (HyperText Markup Language) a été créé en 1991 par Tim Berners-Lee, alors qu'il travaillait au CERN. C'est un langage dit "à balises".

Un langage à balises est un type de langage informatique utilisé pour structurer et organiser des données en les entourant avec des balises ou tags.

Exemple de code HTML

<!DOCTYPE html>
<html>
<head>
    <title>Ma première page</title>
</head>
<body>
    <h1>Bienvenue sur ma page</h1>
    <p>Ceci est un paragraphe.</p>
</body>
</html>

Les balises ne sont pas visibles par l'utilisateur final, elles servent à structurer le document ou à fournir des informations sur la manière dont le contenu doit être rendu.

HTTP : protocole des requêtes sur le WEB

HTTP (HyperText Transfer Protocol) est un protocole de communication qui permet l'échange de données ou de pages sur le WEB. Il fonctionne selon un modèle client-serveur.

Les Méthodes HTTP

Lors d'une communication, le client utilise différentes méthodes pour interagir avec le serveur :

  • GET : utilisée pour récupérer des informations
  • POST : utilisée pour envoyer des données au serveur

Après chaque requête, le serveur renvoie un code de statut qui indique le résultat du traitement :

  • 200 OK : La requête a réussi
  • 404 Not Found : La ressource demandée n'existe pas
  • 500 Internal Server Error : Problème côté serveur

Page Rank

Le Page Rank est un algorithme révolutionnaire développé par Larry Page et Sergey Brin, les fondateurs de Google, lors de leurs études à l'Université Stanford en 1996.

Le fonctionnement du Page Rank est basé sur la logique de "votes". On considère qu'une page vote pour une autre page lorsqu'elle possède un lien vers celle-ci.

Cela veut dire que plus une page possède de liens, plus elle semble "pertinente".

En calculant le nombre de votes, on peut déterminer la qualité d'une page et si elle est pertinente avec la recherche demandée. Cette logique est utilisée par la plupart des moteurs de recherches.
