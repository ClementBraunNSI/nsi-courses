    Fiche d'exercices : Les Dictionnaires en Python

    Danger

    Dans tous les exercices utilisant des dictionnaires, il faut être particulièrement vigilant à : - La vérification de l'existence des clés avec in avant d'y accéder - L'utilisation des bonnes méthodes pour parcourir un dictionnaire : dico.keys(), dico.values(), dico.items() - La distinction entre modification d'un dictionnaire (qui modifie l'original) et création d'un nouveau dictionnaire

    1. Gestion des notes d'élèves

    Un dictionnaire notes contient les noms des élèves en clés et leurs moyennes en valeurs.

    notes = {'Alice': 15, 'Bob': 12, 'Clara': 17, 'David': 10}

    1. Écrire une fonction afficher_notes qui prend en paramètre un dictionnaire et affiche la moyenne de chaque élève sous la forme : "Alice a une moyenne de 15.".
    2. Ajouter un commentaire selon la moyenne : "Excellent" pour une moyenne ≥ 16, "Bien" pour une moyenne entre 12 et 15 inclus, et "À améliorer" pour une moyenne < 12.
    Correction 
      def afficher_notes(notes):
    for eleve in notes:
        moyenne = notes[eleve]
        print(eleve + "a une moyenne de "+ str(moyenne)+".")
        if moyenne >= 16:
            print("Excellent")
        elif 12 <= moyenne <= 15:
            print("Bien")
        else:
            print("À améliorer")
    1. Écrire une fonction ajouter_eleve(notes, nom, moyenne) qui ajoute un nouvel élève au dictionnaire.
    2. Vérifier que l'élève n'existe pas déjà avant de l'ajouter.
    Correction 
      def ajouter_eleve(notes, nom, moyenne):
    # On vérifie d'abord si l'élève existe déjà
    if nom in notes:
        print("L'élève "+nom+" existe déjà !")
    else:
        # Si l'élève n'existe pas, on l'ajoute
        notes[nom] = moyenne
        print("L'élève "+nom+" a été ajouté avec la moyenne de "+str(moyenne)")
    1. Écrire une fonction modifier_moyenne(notes, nom, nouvelle_moyenne) qui modifie la moyenne d'un élève.
    2. Gérer le cas où l'élève à modifier n'existe pas.
    Correction 
      def modifier_moyenne(notes, nom, nouvelle_moyenne):
    # On vérifie si l'élève existe dans le dictionnaire
    if nom in notes:
        # Si oui, on modifie sa moyenne
        notes[nom] = nouvelle_moyenne
        print("La moyenne de "+nom+" a été mise à jour à "+nouvelle_moyenne)
    else:
        print("L'élève "+ nom + n'existe pas dans la liste")
    1. Écrire une fonction eleves_mention(notes, seuil) qui renvoie la liste des élèves ayant une moyenne ≥ seuil.
    2. Afficher également le nombre total d'élèves ayant cette mention.
    Correction 
    def eleves_mention(notes, seuil):
    # On crée une liste vide pour stocker les élèves
    eleves_avec_mention = []
    # On parcourt le dictionnaire
    for eleve in notes:
        if notes[eleve] >= seuil:
            eleves_avec_mention.append(eleve)
    # On affiche le résultat
    print(str(len(eleves_avec_mention))+ "élèves ont une moyenne supérieure ou égale à" + str(seuil)")
    return eleves_avec_mention

    2. Gestion d'un concessionnaire automobile

    Un dictionnaire voitures contient les modèles en clés et leurs prix en valeurs. 

    voitures = {'Clio': 15000, 'Megane': 20000, 'Talisman': 35000}

    1. Écrire une fonction afficher_voitures(voitures) qui affiche le prix de chaque modèle avec le texte : "Le prix de la Clio est de 15000 euros.".

    2. Vérifier que le modèle n'existe pas déjà avant de l'ajouter.
    Correction

    4. Réseau de bibliothèques

    Un dictionnaire bibliotheques contient plusieurs bibliothèques. 

    bibliotheques = {
    'biblio1': {'nom': 'Bibliothèque Centrale', 'ville': 'Paris', 'livres': 30000},
    'biblio2': {'nom': 'Médiathèque', 'ville': 'Lyon', 'livres': 15000},
    'biblio3': {'nom': 'Bibliothèque Universitaire', 'ville': 'Marseille', 'livres': 50000}
}

    1. Écrire une fonction afficher_bibliotheques(bibliotheques) qui affiche les informations sous la forme : "Bibliothèque Centrale à Paris : 30000 livres.".

    2. Afficher uniquement les bibliothèques ayant plus d’un certain nombre de livres.

    3. Écrire une fonction ajouter_bibliotheque(bibliotheques, nom, ville, livres) qui ajoute une nouvelle bibliothèque au dictionnaire.

    4. Vérifier si une bibliothèque avec le même nom existe déjà.

    5. Écrire une fonction plus_grande_bibliotheque(bibliotheques) qui renvoie la bibliothèque ayant le plus de livres.

    6. Modifier la fonction pour qu’elle renvoie également la bibliothèque avec le moins de livres.

    7. Écrire une fonction total_livres(bibliotheques) qui calcule et renvoie le nombre total de livres.

    8. Afficher également la moyenne de livres par bibliothèque.

    5. Base de données des employés

    Un dictionnaire employes contient les informations des employés. 

    employes = {
    'emp1': {'nom': 'Alice', 'poste': 'Développeur', 'salaire': 3500},
    'emp2': {'nom': 'Bob', 'poste': 'Designer', 'salaire': 3000},
    'emp3': {'nom': 'Clara', 'poste': 'Chef de projet', 'salaire': 5000}
}

    1. Écrire une fonction afficher_employes(employes) qui affiche les informations sous la forme : "Alice est Développeur et gagne 3500 euros.".

    2. Afficher uniquement les employés ayant un salaire supérieur à un seuil.

    3. Écrire une fonction ajouter_employe(employes, id_employe, nom, poste, salaire) qui ajoute un nouvel employé au dictionnaire.

    4. Vérifier que l’id_employe n’existe pas déjà.

    5. Écrire une fonction augmenter_salaires(employes, pourcentage) qui augmente tous les salaires d’un pourcentage donné.

    6. Ajouter une option pour n’augmenter les salaires que des employés ayant un poste spécifique.

    7. Écrire une fonction employe_salaire_max(employes) qui renvoie l’employé ayant le salaire le plus élevé.

    8. Modifier la fonction pour renvoyer également l’employé ayant le salaire le plus bas.

    6. Livre de contacts téléphoniques

    Un dictionnaire contacts contient les informations personnelles des contacts. 

    contacts = {
    'Jean Dupont': {'téléphone': '0612345678', 'email': 'jean.dupont@email.com', 'ville': 'Paris'},
    'Marie Martin': {'téléphone': '0687654321', 'email': 'marie.martin@email.com', 'ville': 'Lyon'}
}

    1. Écrire une fonction rechercher_contact(contacts, nom) qui renvoie les détails d'un contact par son nom.

    2. Gérer le cas où le contact n'existe pas avec un message approprié.

    3. Écrire une fonction ajouter_contact(contacts, nom, telephone, email, ville) qui ajoute un nouveau contact.

    4. Vérifier que le contact n'existe pas déjà.

    5. Lever une exception si le contact existe.

    6. Écrire une fonction contacts_par_ville(contacts, ville) qui renvoie la liste des contacts d'une ville donnée.

    7. Retourner une liste vide si aucun contact n'est trouvé.

    8. Écrire une fonction mettre_a_jour_contact(contacts, nom, champ, nouvelle_valeur) qui permet de modifier un champ spécifique d'un contact.

    9. Gérer les cas où le contact ou le champ n'existe pas.

    7. Analyse des résultats d'un sondage

    Un dictionnaire resultats_sondage représente les réponses à un sondage. 

    resultats_sondage = {
    'Satisfait': 45,
    'Neutre': 30,
    'Insatisfait': 25
}

    1. Écrire une fonction calculer_pourcentages(resultats_sondage) qui convertit les nombres en pourcentages.
    2. Arrondir les pourcentages à deux décimales.

    3. Vérifier que le total des résultats est bien de 100%.

    4. Écrire une fonction comparer_sondages(sondage1, sondage2) qui compare deux sondages.

    5. Calculer les différences de pourcentage pour chaque catégorie.

    6. Identifier la catégorie ayant le plus changé.

    7. Écrire une fonction visualiser_resultats(resultats_sondage) qui génère une représentation textuelle des résultats.

    8. Utiliser des astérisques pour représenter graphiquement les proportions.

    9. Écrire une fonction sondage_plus_representatif(list_sondages) qui identifie le sondage avec l'échantillon le plus grand.

    8. Gestion d'un inventaire de jeux vidéo

    Un dictionnaire jeux stocke des informations sur différents jeux vidéo. 

    jeux = {
    'Mario Kart': {'plateforme': 'Switch', 'genre': 'Course', 'note': 9.2},
    'Zelda': {'plateforme': 'Switch', 'genre': 'Action-Aventure', 'note': 9.7}
}

    1. Écrire une fonction filtrer_jeux(jeux, plateforme=None, genre=None, note_min=0) qui filtre les jeux selon différents critères.
    2. Permettre le filtrage par plateforme, genre, et note minimale.

    3. Retourner un nouveau dictionnaire avec les jeux correspondants.

    4. Écrire une fonction meilleur_jeu_par_genre(jeux) qui renvoie le jeu avec la meilleure note pour chaque genre.

    5. Gérer le cas où un genre n'a qu'un seul jeu.

    6. Écrire une fonction ajouter_jeu(jeux, nom, plateforme, genre, note) qui ajoute un nouveau jeu.

    7. Vérifier que la note est entre 0 et 10.

    8. Empêcher l'ajout de jeux en doublon.

    9. Écrire une fonction calculer_moyenne_notes(jeux) qui calcule la moyenne des notes pour tous les jeux.

    10. Distinguer entre moyenne générale et moyenne par genre.

    9. Suivi des performances sportives

    Un dictionnaire athletes permet de suivre les performances. 

    athletes = {
    'Pierre': {'sport': 'marathon', 'temps': [2.15, 2.18, 2.16]},
    'Sophie': {'sport': 'marathon', 'temps': [2.20, 2.22, 2.19]}
}

    1. Écrire une fonction calculer_moyenne_performances(athlete) qui calcule la moyenne des performances.
    2. Gérer les cas de performances manquantes.

    3. Permettre de lisser les résultats en retirant les valeurs extrêmes.

    4. Écrire une fonction meilleur_temps(athletes, sport) qui trouve l'athlète avec le meilleur temps pour un sport donné.

    5. Gérer le cas où aucun athlète ne pratique le sport.

    6. Écrire une fonction progression_athlete(athlete) qui calcule la progression entre les performances.

    7. Retourner un pourcentage de progression.

    8. Gérer les cas de performances irrégulières.

    9. Écrire une fonction ajouter_performance(athletes, nom, sport, temps) qui ajoute une nouvelle performance.

    10. Créer un nouvel athlète si nécessaire.
    11. Vérifier la cohérence des données.
